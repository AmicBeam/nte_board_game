本文只整理动作数据供验收，不修改运行代码、不修改 `actions.json`。

## 整理规则

- 基础动作数据源：`app/modules/shaft/static/data/characters.json` 与 `app/modules/shaft/static/data/actions.json`。
- 普攻数值补充源：腾讯文档 `角色技能数值`（sheet\_id `000003`），只补本文已经存在且能稳定匹配的角色/动作，不新增表中额外动作。
- Q 耗能补充源：`异环云配队.xlsx` 的 `角色` 页第 32 列 `大招消耗`；只有动作名为 `q` 的终结本体消耗能量，其余 Q 变种统一按 `耗能：0` 记录。
- 普攻动作名包含 `5a` 的条目，在本文中拆成 `a1`、`a2`、`a3`、`a4`、`a5` 五个拟动作；娜娜莉 `3z` 与 `3z副手` 分别拆成 `z1-z3` 与 `z1副手-z3副手`。
- 已匹配普攻的备注中，“腾讯表”数值按 `倍率呈现方式` 聚合多段 hit，伤害倍率取 `1级倍率`。
- 拆分后的单段时间仍不硬填；若原 `5a` 有总耗时，时间列写“待拆（原合计 xs）”。
- 动作轴轨道按本次口径标注：普攻链第 1 段（`a1`）固定前台；普攻链第 2-5 段（`a2-a5`）标为 `前台可后台`，这是唯一允许玩家在动作轴里切成后台的普通动作类型；显式后台伤害仍按后台伤害记录。后台无时间动作与 `Q/q` 无时间动作的时间列统一写 `0秒`。
- 安魂曲表额外记录 `噩梦层数`：层数等于该动作伤害段数，`e` 与 `q` 固定 5 层，后台动作记 0；`前台可后台` 动作切到后台时层数不变。
- 卡厄斯表额外记录 `回复罪状`：`a1-a5` 分别为 22、48、28、67、135；下落为 30；`e` 与闪反为 100。
- “原动作”列为 `-` 表示未拆分，直接沿用当前动作名。
