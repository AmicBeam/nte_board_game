# Shaft Buff Nanoka 校对验收清单

本文以 Nanoka 1.2 数据为 buff 配置验收口径。当前实现状态只用于标记“已对齐 / 部分对齐 / 待实现”，具体数值、持续时间、层数和触发条件均以 Nanoka 校对后的规则为准。

校对来源：

- `https://nte.nanoka.cc/`
- `https://static.nanoka.cc/manifest.json`
- `https://static.nanoka.cc/nte/1.2/zh/weapon/{id}.json`
- `https://static.nanoka.cc/nte/1.2/zh/console.json`
- `https://static.nanoka.cc/nte/1.2/zh/character/{id}.json`

说明：

- 装备合法性是所有弧盘 buff 的前置条件：角色类型必须匹配弧盘类型，非匹配组合不能携带，也不参与 buff 与伤害验收。
- `Q` 对应站点「极轨终结」，`E` 对应「变轨技能」，`援护` 对应「援护技」。
- 持续时间单位为秒；最大层数通常为 1，只有 Nanoka 明确写出叠层时才大于 1。
- `action_hit` 按“命中后触发”处理，不影响本次命中，影响后续结算点。
- 敌方 debuff 统一挂在敌人身上：`延滞`、`黯星` 初始 5 秒；`浸染`、`覆纹` 持续 12 秒；`浊燃` 持续 15 秒。
- 真红 `龙a1`、`龙a2`、`龙a3`、`龙a4`、`龙a5`、`龙e`、`穿梭`、`q2` 的 +30% 动作自增伤来自 xlsx 动作行 `self_modifiers`，不是持续 buff，不进入 `buffs.json`。

| 类别 | 已对齐 | 部分对齐 | 待实现 | 说明 |

高优先级待改：

1. `行进于时间之外`、`最后一朵玫瑰`、`极速之棉`、`时间大盗`、`现实避难所`、`危险游戏`。
2. `失落光芒`、`迪亚波罗斯`、`恶魔之血：诅咒`、`森林萤火之心`、`真红：双生蝶` 的作用对象/触发范围需要精确化。
3. 角色被动/觉醒需要新增角色来源 provider，优先处理真红、翳、薄荷、海月、卡厄斯、哈索尔。

## 验收关注点

1. 验收数值、持续时间、层数和触发条件只看 Nanoka 校对后规则。
2. 不把现有实现表当作正确答案；实现状态只用于标记差距和验收进度。
3. 单层 buff 的 `max_stacks` 应为 1；Nanoka 明确多层的条目按上表最大层数配置。
4. `triggered_buffs` 应显示当前动作触发出的 buff，`applied_buffs` 应显示当前动作实际吃到的 buff。
5. 弧盘必须满足角色类型匹配后才可携带和结算；非法角色 + 弧盘组合不作为验收依据。
